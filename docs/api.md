# Students API

The Students API allows the application to create, read, update, and delete students.

Base endpoint:

```txt
/api/students
```

---

## Student Object

A student has the following structure:

```json
{
  "id": 1,
  "firstName": "Lionel",
  "lastName": "Nanseu",
  "dateOfBirth": "2000-01-15",
  "email": "lionel@example.com",
  "phoneNumber": "017600000000"
}
```

| Field | Type | Description |
|---|---|---|
| id | Long | Unique student identifier generated by the database |
| firstName | String | Student first name |
| lastName | String | Student last name |
| dateOfBirth | Date | Student birth date, format `YYYY-MM-DD` |
| email | String | Student email address |
| phoneNumber | String | Student phone number |

# Endpoints

## 1. Get all students

Returns the list of all students.

```http
GET /api/students
```

### Response Example

```json
[
  {
    "id": 1,
    "firstName": "Lionel",
    "lastName": "Nanseu",
    "dateOfBirth": "2000-01-15",
    "email": "lionel@example.com",
    "phoneNumber": "017600000000"
  }
]
```

### Status Codes

| Code | Meaning |
|---|---|
| 200 OK | Students returned successfully |

---

## 2. Get student by id

Returns a single student by its id.

```http
GET /api/students/{id}
```

### Path Parameters

| Name | Type | Required | Description |
|---|---|---|---|
| id | Long | Yes | Student id |

### Example

```http
GET /api/students/1
```

### Response Example

```json
{
  "id": 1,
  "firstName": "Lionel",
  "lastName": "Nanseu",
  "dateOfBirth": "2000-01-15",
  "email": "lionel@example.com",
  "phoneNumber": "017600000000"
}
```

### Status Codes

| Code | Meaning |
|---|---|
| 200 OK | Student found |
| 404 Not Found | Student not found |

---


```json
  "firstName": "Lionel",
  "lastName": "Nanseu",
  "dateOfBirth": "2000-01-15",
```

| Code | Meaning |
|---|---|

## 5. Delete student

Deletes an existing student.

```http
DELETE /api/students/{id}
```

### Path Parameters

| Name | Type | Required | Description |
|---|---|---|---|
| id | Long | Yes | Student id |

### Example

```http
DELETE /api/students/1
```

### Response Body

No response body.

### Status Codes

| Code | Meaning |
|---|---|
| 200 OK | Student deleted successfully |
| 404 Not Found | Student not found |

> Later, this endpoint should return `204 No Content`.

# Notes

## Current implementation

The current implementation uses:

- Spring Boot
- Spring Data JPA
- PostgreSQL
- Hibernate
- REST Controller
- Service Layer
- Repository Layer

---

## Current limitations

The current API does not yet include:

- DTOs
- Request validation
- Global exception handling
- Custom error responses
- Authentication
- Authorization
- Pagination
- Search/filtering

These improvements will be added in later phases.
